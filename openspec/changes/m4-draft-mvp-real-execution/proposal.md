# Change: M4 Draft-MVP Real Execution

## Why

M4's completed capabilities prove governed facts, payload identity, and fake
simulation, but deliberately create no real Docker execution or GitHub Draft
PR. This change defines the separately authorized path to test the registered
fixture environment without expanding to arbitrary repositories or mutation.

## Scope

- a future-only, mutation-scoped fresh PDR and real-mutation request contract;
- one exact local-Docker fixture execution path using the registered image and
  independently verified base-SHA snapshot;
- one fixture-only Draft PR path with runtime-injected opaque credential;
- idempotency, bounded terminals, reset, and redacted audit evidence.

## Non-goals

- arbitrary source, repository, command, image, credential, branch, PR type,
  merge, automatic retry, network-enabled sandbox, or mutation outside the
  registered fixture repository.
- reuse, deserialization, or elevation of `MaterializationPDR`,
  `PayloadEligibilityPDR`, or `forgeflow-sim-*` identity as mutation authority.
